# RAG Lab: Relevance Scoring & Rerankers

Advanced RAG system implementation with reranking for improved retrieval quality on legal documents and podcast transcripts.

## What This Project Does

Implements a complete **RAG (Retrieval-Augmented Generation)** system with these components:

1. **Document Loading & Preparation** - Extracts and structures documents with metadata
2. **Embeddings** - Converts text to numerical vectors using OpenAI
3. **Vector Search** - Stores vectors in Pinecone for fast similarity search
4. **Baseline Retrieval** - Searches documents by embedding similarity
5. **Reranking** - Uses Cohere to re-evaluate results and improve ranking
6. **Evaluation** - Compares results with and without reranking

## How It Works (Simple Explanation)

```
User asks question
    ↓
Embed the question (OpenAI)
    ↓
Search Pinecone for similar documents (5 results)
    ↓
[WITHOUT RERANKING] Return top 5 ← OR → [WITH RERANKING] Send to Cohere
                                             ↓
                                        Cohere re-evaluates relevance
                                             ↓
                                        Return top 3 (best first)
    ↓
User gets better results!
```

## Understanding the Code

### Main Functions

```python
# 1. Convert text to embeddings
get_embeddings(texts) → List[List[float]]

# 2. Search similar documents
basic_search(query, top_k=5) → List[Results]

# 3. Rerank results with Cohere
rerank_results(query, results, top_k=3) → List[RerankedResults]

# 4. Complete pipeline
rag_pipeline(query, use_reranking=True) → Final results
```

### Key Concepts

- **Embeddings**: Text → numbers that represent meaning
- **Vector DB**: Fast searching through millions of embeddings
- **Similarity Score**: 0-1, how similar query is to document
- **Reranking**: Re-evaluating results for actual relevance
- **Metadata**: Extra info (page, section, type) attached to documents

## Performance Notes

- ⏱️ First run: ~2-3 minutes (generating embeddings)
- ⏱️ Subsequent searches: ~1-2 seconds
- ⏱️ With reranking: +1-2 seconds per query
- 💰 Costs: ~$0.02 per 1M embeddings, ~$0.01 per Cohere rerank call

## Troubleshooting

### "API key not found"
- Check `.env` file exists in same directory
- Verify keys are correct format
- Keys should NOT have quotes

### "Pinecone index not found"
- Create index first in Pinecone console
- Or modify code to auto-create index
- Make sure you're using correct index name

### "Embeddings are slow"
- Normal for first run (depends on document count)
- Consider using `text-embedding-3-small` (faster/cheaper)
- For production: batch requests

### "Reranking gives unexpected results"
- Try different test queries
- Increase top_k_search before reranking
- Check Cohere API is working
